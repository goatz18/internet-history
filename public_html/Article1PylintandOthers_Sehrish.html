Pylint and some Tips & Tricks

What is it and why should we use it?

By definition according to pylint.org, "Pylint is a tool that checks for errors in Python code, tries to enforce a coding standard and looks for code smells. It can also look for certain type errors, it can recommend suggestions about how particular blocks can be refactored and can offer you details about the code's complexity."

    Some important definitions!

  • Factory Method: it is a creational design pattern which solves the problem of creating product objects without specifying their concrete classes.
  • Instantiation: The creation of an instance of a class.
  • Object: A unique instance of a data structure that's defined by its class.
  • Class: A user-defined prototype for an object that defines a set of attributes that characterize any object of the class. The attributes are data members and methods, accessed via dot notation.
  • Constructor: Code that is executed each time an instance of the class is created.
  • Static class: "A static class can be used as a convenient container for sets of methods that just operate on input parameters and do not have to get or set any internal instance fieldsClass that cannot be instantiated, so no objects can be created from the class. The class members are instead accessed using the class name. A static class provides a shortcut to the members of the class when instancing is unwarranted." --docs.microsoft.com
  • Methods: Operations (or actions) that objects perform or operations which are performed to an object. Sometimes referred to as behaviors.
  • Namespace: A namespace is a system that has a unique name for each and every object in Python. An object might be a variable or a method.
  • Fixture: Fixtures define the steps and data that constitute the arrange phase of a test (see Anatomy of a test).
  • Type Hint: "Type hinting gives you the ability to define the type of values that can be passed for each argument of a function or method."-https://torquemag.io/2016/09/wordpress-developers-guide-type-hinting/
  • Type Cast: "Type casting is a method used to change the variables/ values declared in a certain data type into a different data type to match the operation required to be performed by the code."--https://techterms.com/definition/typecasting
  • Facade: Facade is a structural design pattern that provides a simplified (but limited) interface to a complex system of classes, library or framework.
  • Property: The property() method in Python provides an interface to instance attributes
